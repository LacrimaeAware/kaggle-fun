## Later: the engine is public, and the first agent is measured

Big finding from the research workflow + verification: the engine is the `cabt` env in the
open-source `kaggle_environments` package. Installed it, it runs locally:
`make("cabt").run([agent, agent])`. Agent API is `agent(obs) -> list[int]`: deck phase
returns 60 card IDs, play phase returns indices into `obs["select"]["option"]`.

Read the engine (`cg/sim.py`, `cg/game.py`): only `BattleStart`/`Select`/`GetBattleData`
on one global native battle, no clone/snapshot, no search API. So H1 (in-match forward
model for tree search) is refuted. Offline self-play works, so a learned eval trained
offline is still viable; in-match ISMCTS is not, this version.

Built `main.py` (always-legal heuristic, deck embedded) and measured in the REAL engine:
- heuristic vs random 0.835, first_agent vs random 0.830, heuristic vs first 0.515.
- So the type-14 "defer the end-turn" idea ties take-first (H16 refuted). The win is
  consistency over random, not the heuristic. Logged honestly to the registry.

Prize question answered: the "$240k" is Round-1 Strategy top 8 x $30k each. Round 2 adds
$50k/$30k + GC credits. Simulation track is $0 ("Knowledge"). Live Rules page is the
authority on the exact total.

## Even later: audited the game + downloaded reference data

Audited the real game (audit_game.py). Concrete findings:
- The observation is rich: each Pokemon shows hp/maxHp/energies/energyCards/preEvolution
  (evolution stack)/tools/status flags; each player shows active, bench (max 5), deckCount,
  discard, 6 hidden prizes, handCount, own hand. So a board-aware eval has every feature.
- Option types decoded: 13 = attack (attackId), 14 = end turn, 8 = target an in-play
  Pokemon, 3 = select a card, 7 = pick from a list, 1/2 = menu choices.
- Games terminate reliably: random-vs-random median 57 engine steps, max 263, ZERO draws.
  No infinite-loop problem. First-player win rate 0.453 (no first-move edge).
- Card data (attack damage/cost) is NOT in the Python layer (it is in the native cg.dll),
  but is recoverable by logging defender HP before/after each attack in self-play.

Downloaded reference data into data/external/ via tools/fetch_data.py:
- 16 SV-era card sets (3250 cards) from the pokemon-tcg-data GitHub repo.
- 1630 real Standard decklists with placings from the Limitless play API.
- Caveat: real card names, but the cabt engine uses its own integer card IDs and an
  organizer card pool, so this is human strategy reference, not a cabt mapping.
