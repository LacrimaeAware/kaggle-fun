# journal/

Daily brain-dump. One file per day, named `YYYY-MM-DD.md`. Unlimited files: this is the
one place where making a new document every day is correct.

What goes here: what was tried, what was seen, half-formed ideas, dead ends, questions,
links to chase. It is allowed to be messy, repetitive, and wrong. It is not
authoritative and nothing reads it as truth.

What does not go here: a conclusion you want future sessions to trust. Those graduate
out of the journal into a falsifiable hypothesis in the registry
(`python ../registry/registry.py add ...`) or into a curated doc in `../docs/`. The
journal is the intake; the registry and `docs/` are the curated layers.

Rule of thumb: if you find yourself writing the same conclusion into a third day's
journal, that is the signal to make it a hypothesis and test it, or to check whether it
is already in `../registry/GRAVEYARD.md`.
