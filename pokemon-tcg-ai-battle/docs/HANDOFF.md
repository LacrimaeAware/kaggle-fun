## The reframe (do not re-inherit the old framing)
A model trained PURELY to clone teacher T cannot beat T. The mistake was fixing T = the live N=8 search
and concluding learning is capped. T can instead be a SLOW OFFLINE search (N=32/64, deeper on hard
decisions, robust aggregation) far stronger than the live agent; and expert-iteration lets the learned
policy guide a stronger planner than either alone. That is the entire point of the branch split. Do not
call the learner "capped at search." (Caveat per plan point 6: terminal OUTCOME is an auxiliary
critic/validation target only, NOT the primary action target -- the primary target stays repeated
counterfactual action advantage from the teacher.)

## Teacher primitives that ALREADY EXIST (Branch A starts here, do not rebuild)
- `agent/search.py: option_evals(obs, deck, time_budget)` -> per legal option, (mean leaf value, mean leaf
  features) averaged over N_DETERM determinizations. This is the per-action counterfactual value.
- `option_deltas(obs, deck)` -> per option, one-step consequence dict (prizes/KO/dmg/draw/energy/board...).
- `best_option` / `_search` -> the live move choice; `_search` already computes per-option averages.
- `N_DETERM` (module global), `DEFAULT_BUDGET=0.6` (per-decision cap), `MY_CONT` (rollout policy).
The "teacher API" the plan wants = wrap option_evals to also return value variance, top-2 margin, completed
determinization count, soft policy / normalized advantage, and the semantic equivalence class per option.

## Landmines (the bug class the shared preflight must kill)
- Option card identity depends on TYPE: PLAY (type 7) has NO `area` field, its `index` is a hand index;
  attach/evolve are area==2; ability/attack reference the in-play pokemon. See `build_action_dataset.opt_card_id`
  (already fixed). Get this wrong and embeddings/effects go dead.
- Canonical equivalence: `opt_key` collapses strategically-identical options; two different PLAY options must
  NOT collapse. Score within-decision metrics by eq-class, not raw index.
- Comparing two search configs (e.g. continuation policies) under the 0.6s cap CONFOUNDS quality with
  "how many determinizations finished." Fix the determinization count + raise the cap to isolate quality.
- A "meta/opponent" deck must actually DIFFER from ours (the top corpus deck IS our deck) or the test is a no-op.
- Agents fall back to the heuristic on any exception (silently). Before trusting an arena result, verify the
  thing under test is actually being exercised (we checked the net fired on 60/60 decisions, 0 fallbacks).
- A learned target with no labels silently trains nothing (printed a fake 0.000); assert labels exist.
